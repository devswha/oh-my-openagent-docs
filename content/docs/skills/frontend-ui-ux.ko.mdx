`frontend-ui-ux`는 디자인 스킬입니다. UI/UX, 스타일링, 애니메이션, 레이아웃 작업 시 로드 — 보통 브라우저 검증용 `playwright`와 함께.

## 트리거

UI/UX 작업, 스타일링, "design", "responsive", "이거 더 예쁘게".

## 디자인 프로세스

1. **Purpose** — 이 화면이 뭘 위한 것인가.
2. **Tone** — brutalist, maximalist, retro-futuristic, luxury, playful — 극단을 골라라.
3. **Constraints** — 고정된 것 (브랜드, 치수, 지원 브라우저).
4. **Differentiation** — 무엇이 이걸 구별 짓는가.

## 스킬이 피하는 안티 패턴

- 평범한 폰트 (Inter, Roboto, Arial).
- 흰 배경 위 보라 — 전형적 "AI slop" 팔레트.
- 예측 가능한 레이아웃, 쿠키 커터 디자인.
- "그냥 페이드" 호버 상태.

## 조합

- `playwright` — done 선언 전 실제 브라우저에서 렌더 검증.
- `task(category: "visual-engineering")` — 구현은 기본 Gemini 3.1 Pro로 실행.
