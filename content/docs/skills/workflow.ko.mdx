## 주요 워크플로우

| Workflow | 목적 |
|---|---|
| `ulw` / `ultrawork` | Sisyphus가 알아서 탐색, 구현, 검증까지 미는 자율 루프. |
| `ralph-loop` / `ulw-loop` | 실행과 검증을 반복하는 self-continuing 루프. |
| `@plan` / Prometheus mode | 구현 전에 인터뷰로 계획을 작성. |
| `/start-work` | 최신 Prometheus 계획을 Atlas로 실행. |
| `init-deep` | 계층형 프로젝트 지식 빌드. |
| `handoff` | 다음 세션을 위한 문맥 보존. |
| `refactor` | 진단, AST-Grep, 동작 보존 검증이 들어간 리팩터링 레인. |

## 선택 기준

자율성이 중요하면 `ulw`. 오래 남는 계획과 명시적 검증이 필요하면 Prometheus와 `/start-work`.
