---

구현 에이전트가 작업 중에 제품이나 아키텍처 결정을 즉흥으로 만들면 안 됩니다. 이 셋은 코드가 바뀌기 전에 그 결정이 내려지고 — 검토되도록 합니다.

## 계획 루프

| 단계 | Agent | 산출물 |
|---|---|---|
| 명확화 | Prometheus | 인터뷰 질문, 가정, 범위 경계. |
| 갭 분석 | Metis | 숨은 요구사항, 모호함, anti-slop 지시. |
| 계획 작성 | Prometheus | 작업과 검증이 들어간 `.sisyphus/plans/*.md`. |
| 정밀 리뷰 | Momus | blocker 기준 `OKAY` 또는 `REJECT`. |
| 실행 | Atlas | 위임된 작업과 누적 학습. |

## 언제 쓰나

리팩터링, 아키텍처 결정, 여러 파일에 걸친 기능, "그냥 구현"하면 우연에 맡기는 게 너무 많아지는 일에 이 레인을 부릅니다.
