세션 도중에 찾아보는 것들을 모은 평면 레퍼런스입니다. 명령 이름, config 경로, 에이전트 옵션, install flag, 텔레메트리를 끄는 환경 변수.

## 패키지 이름

upstream rename은 전환 중이고 두 패키지가 모두 게시되고 있습니다. `oh-my-openagent`를 기본으로 쓰고, `oh-my-opencode`는 호환 alias로 남깁니다.

| 이름 | 용도 |
|---|---|
| `oh-my-openagent` | upstream에서 선호하는 npm 패키지이자 기본 installer command. |
| `oh-my-opencode` | 이름 변경 전환기 동안에도 함께 게시되는 호환 패키지. |
| `oh-my-openagent` plugin entry | installer가 등록할 수 있을 때 선호되는 OpenCode plugin entry. |
| `oh-my-opencode` plugin entry | 전환기에도 인식되는 legacy entry. `doctor`는 `opencode.json`에 남아 있으면 경고를 띄웁니다. |

## CLI

| 명령 | 용도 |
|---|---|
| `bunx oh-my-openagent install` | 대화형 installer. |
| `bunx oh-my-openagent install --no-tui ...` | agent-driven 설치를 위한 비대화형 setup. |
| `bunx oh-my-openagent doctor` | system, config, tool, model resolution 진단. |
| `bunx oh-my-openagent run <message>` | completion enforcement가 들어간 OpenCode session runner. |
| `bunx oh-my-openagent get-local-version` | 설치된 plugin 버전과 업데이트 상태 확인. |
| `bunx oh-my-openagent refresh-model-capabilities` | models.dev에서 model capabilities 캐시 갱신. |
| `bunx oh-my-openagent version` | 버전 정보. |
| `bunx oh-my-openagent mcp oauth login\|logout\|status ...` | remote MCP 서버용 OAuth helper (PKCE + dynamic client registration). |

## Config 파일

User config가 먼저 load되고, project config가 override할 수 있습니다. JSONC를 지원합니다.

| 범위 | 경로 |
|---|---|
| 프로젝트 | `.opencode/oh-my-openagent.json[c]`, `.opencode/oh-my-opencode.json[c]` |
| 사용자 | `~/.config/opencode/oh-my-openagent.json[c]`, `~/.config/opencode/oh-my-opencode.json[c]` |

## 빠른 Config

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/code-yeongyu/oh-my-openagent/dev/assets/oh-my-opencode.schema.json",
  "agents": {
    "oracle": { "model": "openai/gpt-5.4", "variant": "high" },
    "explore": { "model": "github-copilot/grok-code-fast-1" }
  },
  "categories": {
    "quick": { "model": "opencode/gpt-5-nano" },
    "visual-engineering": { "model": "google/gemini-3.1-pro" }
  }
}
```

설치된 패키지가 renamed schema를 노출한다면 `doctor`가 알려주는 경로를 우선하세요.

## Agent 옵션

| 옵션 | 의미 |
|---|---|
| `model` | 명시적인 `provider/model` override. |
| `variant` | `max`, `high`, `medium`, `low`, `xhigh` 같은 변형. |
| `fallback_models` | 모델/공급자 실패 시 순서대로 시도할 fallback chain. |
| `prompt`, `prompt_append` | agent prompt 교체/추가. `file://` URI 지원. |
| `tools` | 에이전트가 쓸 수 있는 도구를 제한. |
| `disable` | 특정 에이전트 비활성화. |
| `permission` | edit, bash, webfetch 같은 권한 제어. |

## Category

| Category | 목적 |
|---|---|
| `visual-engineering` | Frontend, UI/UX, design, styling, animation. |
| `ultrabrain` | hard logic, architecture, deep reasoning. |
| `deep` | 자율 조사 + end-to-end 구현. |
| `artistry` | 창의적/비관습적 작업. |
| `quick` | 단일 파일 수정, typo fix, 간단한 변경. |
| `unspecified-low`, `unspecified-high` | 일반 저/고 effort fallback 카테고리. |
| `writing` | 문서와 prose. |

## Install Flags

| Flag | 의미 |
|---|---|
| `--claude=no\|yes\|max20` | Claude provider mode. |
| `--openai=no\|yes` | OpenAI/ChatGPT provider mode. |
| `--gemini=no\|yes` | Gemini provider mode. |
| `--copilot=no\|yes` | GitHub Copilot fallback provider. |
| `--opencode-go=no\|yes` | OpenCode Go subscription path. |
| `--opencode-zen=no\|yes` | `opencode/` model catalog path. |
| `--zai-coding-plan=no\|yes` | Z.ai Coding Plan path. |
| `--kimi-for-coding=no\|yes` | Kimi for Coding path. |
| `--vercel-ai-gateway=no\|yes` | Vercel AI Gateway path. |
| `--skip-auth` | 설치 후 auth 안내 생략. |

## Environment

| 변수 | 의미 |
|---|---|
| `OMO_SEND_ANONYMOUS_TELEMETRY=0` | 익명 active-install telemetry 끄기. |
| `OMO_DISABLE_POSTHOG=1` | PostHog telemetry 끄기. |
| `OPENCODE_CONFIG_DIR` | OpenCode config directory override. |
