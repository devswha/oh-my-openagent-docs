## Background Agents

Agents can run in parallel while the main session keeps working. This is how OmO searches, researches, and verifies without bloating one context.

## LSP and AST-Grep

OmO exposes IDE-grade operations: diagnostics before build, go-to-definition, references, workspace rename, and AST-aware search/replace across supported languages.

## Tmux

Interactive shell sessions, REPLs, debuggers, and long-running commands can run in tmux panes. Agents can drive the terminal instead of pretending source inspection is enough.

## MCPs

Web search, Context7 documentation, GitHub code search, and skill-embedded MCP servers are available when the task needs them.

## Hashline Edits

Hashline read/edit surfaces attach stable content IDs to lines. Edits can be rejected when the file changed after it was read, reducing stale-line corruption.
