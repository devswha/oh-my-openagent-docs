## 백그라운드 에이전트

Explore, Librarian 같은 에이전트를 백그라운드로 띄우면 메인 세션은 구현을 계속하면서 조사 결과를 나중에 받을 수 있습니다. OmO의 속도는 여기서 많이 나옵니다.

## LSP와 AST-Grep

| 도구 | 쓰임 |
|---|---|
| LSP diagnostics | 빌드 전에 타입/언어 서버 오류를 확인합니다. |
| goto definition / references | 심볼 정의와 사용처를 추적합니다. |
| workspace rename | 이름 변경을 프로젝트 전체에 적용합니다. |
| AST-Grep | 문자열 검색이 아니라 AST 패턴으로 찾고 바꿉니다. |

## Tmux와 실제 표면 검증

CLI, TUI, REPL, 디버거는 tmux에서 직접 실행할 수 있습니다. 웹 UI는 브라우저로 열고 클릭해야 합니다. HTTP API는 실제 프로세스에 요청을 보내야 합니다. OmO는 “소스만 읽고 될 것 같다”를 완료로 보지 않습니다.

## MCP와 검색

- **websearch**: Exa 기반 웹 검색.
- **context7**: 공식 문서와 API 사용 예시.
- **grep_app**: GitHub 코드 검색.
- **skill-embedded MCP**: 스킬이 필요한 도구를 필요할 때만 로드합니다.

## 해시라인 편집

Hashline은 읽은 줄에 안정적인 콘텐츠 ID를 붙입니다. 파일을 읽은 뒤 다른 프로세스가 바꿨다면, 잘못된 줄을 수정하기 전에 편집을 거부할 수 있습니다. 여러 에이전트가 동시에 움직이는 환경에서 특히 중요합니다.

## Hook과 Continuation

공개 설정 레퍼런스에는 40개 이상의 lifecycle hook이 나옵니다. 핵심은 세 가지입니다.

| 계열 | 역할 |
|---|---|
| session recovery | 손상된 세션/메시지 상태를 복구합니다. |
| todo continuation | 미완료 todo가 남았는데 멈추는 것을 막습니다. |
| background notification | 백그라운드 작업 완료/실패를 메인 세션에 알려줍니다. |
